Connecting to the backend and receivng the streams

A short overview how to connect the sdk and receive the current liste streams.

Getting started

After you started the SDK and received the RaydioSDKProviding

You can start the connect()function.

But before you do you should register a delegate for the RaydioSDKStateDelegateto receive updates for the state of the sdk.

public protocol RaydioSDKStateDelegate {
    func onRaydioSDKStateChanged(_ state: RaydioSDKState)
}

public enum RaydioSDKState: Equatable {
    case idle
    case connected
    case disconnected
}

Use the sdkStateDelegate to register the delegate.

After you registered the delegate and called the connect function you will receive a state update with either connected or disconnected.

Handling disconnected

Everytime you receive the disconnected state you should reconnect the sdk if possible by calling the reconnect() method.

When do you receive the disconnect state?

We internally use websockets, so whenever the socket is closed. This can happen for different reasons:

  • The app goes into the background

  • The network connection is lost

  • A problem establishing the connection to our backend

You need to observe on you own the network and if the app is in foreground/background. See

If the app is in background you need to wait until it moved to the foreground to reconnect.

If the network was lost, you need to wait until the network was regained.

Both are further described here

Handling connected

You now have established the connection to our backend and can start requesting the current live streams:

For that call the requestStreams() function from the sdk.

Before doing this you should register the delegate that will acatually receive the streams and updates for those streams StreamGroupDelegate

Descriptions of the different functions are provided in the comments. 

import Foundation

/// Implement this delegate to receive updates when a streams are changed
public protocol StreamGroupDelegate {

    /// A new group was added, this is the first stream for a specific game
    ///
    /// - Parameters:
    ///     - group: The newly added group (The first stream for this title))

    func onNewGroup(_ group: StreamGroup)

    /// A group was updated, this can either mean, an additional stream was added, removed or that the state of
    /// a stream changed (for example the mute state)
    ///
    /// - Parameters:
    ///     - group: The updated group
    func onGroupUpdated(_ group: StreamGroup)

    /// A group was removed. This means there is no more stream online for the title of the group
    ///
    /// - Parameters:
    ///     - group: The group that was removed
    func onGroupRemoved(_ group: StreamGroup)

    /// We received the currently available streams. This is done when the connection to the server was established or the
    /// Streams are requested again
    ///
    /// - Parameters:
    ///     - groups: The list of groups
    func onGroupsReceived(_ groups: [StreamGroup])
}

To to this you need to get the StreamsProviding protocol and on that you can register the delegate on the delegate property.

/// Get access to the streams and you can register for updates of the streams here
public protocol StreamsProviding {
    /// List of current stream groups
    var groups: [StreamGroup] { get }

    /// The delegate to receive updates for changes within the stream groups
    var delegate: StreamGroupDelegate? {get set}
}

To get the StreamsProviding from the sdk use the raydioStreams property.

Stream Groups

A StreamGroup is a collection of streams that a grouped together by the same match.

/// A stream group represents a collection of all streams belonging to the same match
///
public struct StreamGroup {
    /// The title of the match
    public var title: String

    /// The matchId provided by the backend
    public var matchId: String

    /// All currrently available streams for the match
    public var entries: [StreamGroupEntry]
}

As you can see, there is a list of entries of type StreamGroupEntryand all share the same title (the name of the match, like Denmark vs England).

StreamGroupEntry

A single stream within a group that can be used to listen to from the streamId and contains the required information like the actual language and if the streamer is currently isMuted . The matchId is the same as in the corresponding group.

/// A single entry within a group
/// This entry represents a stream as part of a group of streams belonging to the same match
public struct StreamGroupEntry {
    /// The language of the stream as the native name
    /// e.g for German it would state: Deutsch
    public var language: String

    // The id of the stream
    // this is used for playing and others
    public var streamId: String

    // is the stream currently muted from the streamer side
    public var isMuted: Bool

 // The match id for the game provided by the backend
    public var matchId: String
}

When to request streams

After the intial connection was established you should request the streams once.

The delegate function onGroupsReceived(_:) will be called with all currently available groups.

After that you will receive live updates on the other 3 functions of the delegate as long as the sdk is connected.

This means that you should request streams again when the sdk is potencially coming from a disconnected state.

So when moving the app to the foreground or when network connection has been reastablished.

This can always be done in the state change listener, as the sdk changed back to state connected if you call the appropriate functions.
