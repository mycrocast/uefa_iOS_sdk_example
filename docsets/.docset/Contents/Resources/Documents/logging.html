Logging

Logging can be used to log interactions from the user with the system or general information what is happening internally.

The developer needs to store the logs if wanted because the SDK just forwards them without storing them anywhere.

Log Types

There are in total 4 different log types providing different information:

  • InformationLog

  • ErrorLog

  • SystemErrorLog

  • InteractionLog

Information Log

The information log is just providing general information what the sdk is currently doing. Like “requesting streams”, “received x streams” and so on. This could be useful to see if some steps are missing or are executed in the wrong order.

/// Represents an information about what the sdk is currently doing or trying to do
/// Like "connecting" ...
public struct InformationLog {
    /// The current workflow the sdk is in
    var workflow: WorkflowLogs

    /// a short description 
    var description: String
}

See the comments within the class for more information.

The WorkflowLogs enum looks like this:

/// Enum describing from which workflow the log was generated from
public enum WorkflowLogs {
    case connecting // we are currently in the process of connecting to the backend
    case subscribing // we are subscribing, basically registering in the backend to be allowed to send messages
    case streams // we are requesting the current streams or handling updates from them
    case playing // we are in the workflow of wanting to play and all the steps required to establish a connection with the audio
}

This will help you determine from where the log was produced.

ErrorLog

The errorLog contains the error information that are produced by the sdk.

Actual error handling is described here //TODO Link

This is again helpful for later debugging or providing us with more information in case something goes wrong.

The struct:

/// Represents an error log entry that is thrown by the sdk
/// for a specific workflow and with some kind of description
/// 
public struct ErrorLog {
    /// The workflow at which the error occurred
    var workflow: WorkflowLogs

    /// A plain text description providing more information
    var description: String

    /// The actual error
    var error: RaydioError
}

SystemErrorLog

This log entry is produces when we got a low level error in the system. This is purely for debugging purposes and should actually not happen. An example would be a failed deserialisation of a response.

The struct:

/// Represents a low level system error that was thrown
/// at some point during some workflow
public struct SystemErrorLog {
    /// The workflow this error occured
    var workflow: WorkflowLogs

    /// A short description if possible about the error
    var description: String

    /// The actual error
    var error: Error
}

InteractionLog

The interaction log is produced when there was an interaction with the system.

This is either a play or pause of a stream. This can happen more times than the user actually presses play/pause as internal reconnects (after network issues) also trigger this.

If you want to track the actual user only interactions, you should observe your ui.

The struct:

/// Represents the interaction of play or pause
/// This can be triggered by the user hitting a button but also by the system on a reconnect
public struct InteractionLog {
    /// What happened?
    var type: InteractionType

    /// The stream this interaction has happened on
    var stream: StreamGroupEntry
}


/// enum representing what has happened
public enum InteractionType {
    case play /// a play was triggerred
    case pause /// a pause was triggered
}

How to receive logs

To actually receive the Logs you need to implement the LogDelegate

/// Implement this delegate to receive log information
/// provided by the sdk
public protocol LogDelegate {
    /// called to provide information logs
    func onInformationLog(_ information: InformationLog)

    /// called to provide error logs, this are errors thrown by the sdk itself and
    /// related to some kind of workflow
    func onErrorLog(_ error: ErrorLog)

    /// called to provide logs that a user has performed an interaction
    /// this is limited to play and pause of  a stream
    func onInteractionLog(_ interaction: InteractionLog)

    /// called to provide system errors, these are low level errors that cannot be mapped
    /// to any useful state in the sdk and should not happen
    func onSystemErrorLog(_ systemError: SystemErrorLog)
}

You need to register this class on the logDelegate property of the RaydioSDKProviding
