Observing background state and network

Observing certain aspects of the app state is left to the developer but important for the SDK to deliever the best user experience.

Background/Foreground

You need to observe the background and foreground state of you app and call the corresponding functions from the sdk.

Going to background:

When the app is going into the background call onBackgroundEntered() so that the SDK can free ressources that otherwise would use up battery while not beeing used.

Going to foreground:

When the app is moving to the foreground call onForegroundEntered() so that the SDK can reconnect internally.

Connection state change

You also need to observe the network state change and call the corresponding functions provided by the sdk.

Connection lost

In case the connection was lost you need to call the onConnectionLost() function on the sdk. This will clean up some internal states.

Connection available

If the connection is available again you need to call the onConnectionReestablished() function. This will trigger a reconnect of the sdk and in case that the listener was listening to stream before losing the internet connection, will automatically start the playback again.
